# 安全策略 / Security Policy

## 中文

请不要公开提交尚未修复的安全漏洞。使用 GitHub 仓库的 **Security → Report a vulnerability** 私密报告渠道，并提供受影响包与版本、影响范围、复现步骤和建议修复方案。维护者会尽快确认报告；修复与披露时间将根据风险共同协调。

只有 npm 上仍受支持的最新稳定版本会接收安全修复。请勿在报告中包含真实凭据、个人数据或生产环境密钥。

## English

Do not open a public issue for an unpatched vulnerability. Use the repository's private **Security → Report a vulnerability** channel and include the affected package and version, impact, reproduction steps, and a suggested remediation. Maintainers will acknowledge the report as soon as practical; remediation and disclosure timing will be coordinated according to risk.

Only the latest stable versions still supported on npm receive security fixes. Do not include real credentials, personal data, or production secrets in a report.
