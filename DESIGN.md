# ZhuàngTài 文档站设计系统

## 0. 当前依据

本文件描述当前文档站的真实设计约束。最终判断以生产构建中的浏览器计算样式、交互结果和无障碍树为准。

## 1. 产品气质

ZhuàngTài 的界面应与状态原语一致：简单、直接、可预测。文档内容优先，在线示例应像一段可直接上手的小应用。

- 不使用装饰性渐变、无意义动效或与内容无关的视觉噪声。
- 通过明确层级、留白、语义边框和少量强调色建立结构。
- 示例先展示用户能完成的任务，再展示对应核心代码。

## 2. 技术与颜色

- 文档自定义界面统一使用 Tailwind CSS v4 utilities。
- 不新增组件级 `<style>`、内联 `style=` 或传统手写样式表。
- 唯一 CSS 入口只负责导入 Tailwind theme 与 utilities。
- 文档外壳和组件色彩使用 Starlight 的 `--sl-color-*` 语义变量，自动适配浅色和深色主题。
- 示例内部模拟的应用预览可以使用一组封闭的演示色，但不得影响文档全局主题。

## 3. 排版与空间

- 页面只保留一个一级标题，内容章节从二级标题开始。
- 正文继承 Starlight 字体；代码和面板标签使用文档等宽字体。
- 在线示例页关闭右侧页内导览，把可用宽度交给实时预览与代码。
- 宽屏示例容器可突破正文的 `45rem` 阅读宽度，但不能超过主内容区或造成横向滚动。
- 移动端恢复单栏，控件不得被裁切。

## 4. 交互控件

- 主要按钮、次要按钮、示例标签、筛选按钮和文本输入统一为 `2.5rem` 高。
- 同一按钮组内宽度由网格统一；图标按钮可固定宽度，但高度保持一致。
- 圆角层级：控件 `0.5rem`，内部卡片 `0.75rem`，主要容器 `1rem`。
- 主要动作使用强调色实底和高对比文本；次要动作使用语义边框与页面背景。
- 所有控件必须有清晰的 hover、disabled、pressed 和 `focus-visible` 状态。

## 5. React 在线示例

在线示例由真实 `@zhuangtai-js/react` 组件驱动，并按职责拆分：

- 容器负责示例选择、键盘导航、预览与代码布局。
- Counter、Tasks、Preferences 各自拥有独立模块和状态。
- 文案、代码片段、类型与共享控件样式分别独立维护。
- 单个组件文件应保持聚焦，避免重新形成数百行的综合文件。

示例必须允许用户直接观察状态结果：

- Counter：增减、重置与派生值。
- Tasks：输入、添加、完成、筛选、删除与统计。
- Preferences：主题、密度、实时预览与浏览器持久化。

## 6. 响应式

- 小于 `32rem`：示例选择器和任务输入区使用单列。
- 小于 `62rem`：预览与代码使用单列。
- `72rem` 及以上：示例容器在主内容区内扩展，最大宽度 `60rem`，并与正文保持同一视觉中心。
- 任意视口下 `documentElement.scrollWidth` 不得超过 `clientWidth`。

## 7. 无障碍

- 示例选择器保持正确的 `tablist`、`tab`、`tabpanel` 关系。
- 支持 `ArrowLeft`、`ArrowRight`、`Home`、`End`，焦点跟随选中项。
- 分组控件使用 `fieldset` 与 `legend`；动态数值和统计使用礼貌播报。
- 文本对比度达到 WCAG AA，反馈不能只依赖颜色。
- SSR 输出至少包含默认示例内容；JavaScript 水合后提供完整交互。
- 遵循 `prefers-reduced-motion`，不以动画延迟或掩盖同步状态更新。
